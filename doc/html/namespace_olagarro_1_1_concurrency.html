Launches a function concurrently taking into account passed element number and host's CPUs.

Launches function function in as much batches as CPUs has host machine. For example, if host has 4 CPUs it will divide [begin, end) range in 4 parts and launch a concurrent task for each part. As other concurrency module facilities user is responsible to avoiding race conditions and such issues.

Parameters
beginRange's start position iterator
endRange's end position iterator
functionA function which takes range's type's reference as parameter. This reference will be used to change range's current element
Returns
A Future<void>. This object is used to know when concurrentFor has finished its job

template<typename InputIterator , typename Functor >
Future<Functor> Olagarro::Concurrency::concurrentReductorFor ( InputIterator  begin,
InputIterator  end,
Functor  functor 
)

This version executes for concurrently and also applies a merging or reducing method.

This version of concurrentFor functions like previous functor version but also expects the functor has defiend a method called "merge", which accepts an object of functor's own type. The following is an example to find the maximum value of a vector<int>:

struct MaximumFinder
{
MaximumFinder() :
maximum(std::numeric_limits<int>::min())
{}
// This member function will be called for every element of vector<int>
void operator () (int elementIndex, int element)
{
maximum = std::max(element, maximum);
}
void merge(const MaximumFinder& otherMF)
{
maximum = std::max(otherMF.maximum, maximum);
}
int maximum;
};
std::vector<int> v(getVectorDataFromSomewhere());
// Notice how Future is not void but MaximumFinder, same type as our functor
Olagarro::Future<MaximumFinder> forFuture = Olagarro::concurrentReductorFor(v.begin(), v.end(), MaximumFinder());
std::cout << "v's maximum value is = " << forFuture.result() << std::endl;
Parameters
beginRange's start position iterator
endRange's end position iterator
functorA functor with operator () and merge() methods defined (see example). Be aware that functor will be copied many times (each thread has its own copy and so does Future return object)
Returns
A Future<FunctorType> which holds functor's copy

template<typename ReturnType , typename Functor >
Future<ReturnType> Olagarro::Concurrency::launchJob ( const Functor &  functor)

Launches a functor's operator () asynchronously.

As concurrency module only offers 0 or 1 parameter function/method versions, functors versions allow to encapsulate more elaborated calls. For example if we have this function:

float getSomeValue(const std::vector<float>& data, const OtherData& otherData, float thirdValue, std::string& lastData);

We can wrap it in a functor and launch it concurrently:

struct CallWrapper
{
CallWrapper(const std::vector<float>& data, const OtherData& otherData, float thirdValue, std::string& lastData) :
mData(data),
mOtherData(otherData),
mThirdValue(thirdValue),
mLastData(lastData)
{}
float operator () () const
{
return getSomeValue(mData, mOtherData, mThirdValue, mLastData);
}
const std::vector<float>& mData;
const OtherData& mOtherData;
float mThirdValue;
std::string& mLastData;
};
Future<float> myValue = Olagarro::launchJob(CallWrapper(...The 4 parameters...));
Parameters
functorA functor with defined const operator () taking 0 parameters. This operator must return same type of value as Future is expecting (void also accepted)
Returns
A Future which will contain functor's operator () result once it finishes its job
